## Что сделано

В проект добавлен консольный `TaskManager` с операциями:

- `add <task>`
- `remove <task>`
- `list`

Также реализована централизованная обработка исключений через `ExceptionHandler`.

## Что логируется

При любом перехваченном исключении фиксируются:

- текст ошибки (`Exception.Message`);
- стек вызовов (`Exception.StackTrace`);
- контекст выполнения (какая операция выполнялась);
- уровень ошибки (`Warning`, `Error`, `Fatal` и др.).

Логи сохраняются в файл:

- `logs/errors.log`

## Механизм оповещения

Добавлен базовый механизм оповещения:

- вывод в консоль сообщения `!!! Ошибка !!!`;
- запись факта оповещения в лог.

Для серьезных ошибок уровня `Fatal` предусмотрен отдельный репортер `ExternalIncidentReporter`.
Сейчас он реализован как заглушка и показывает, где подключать:

- Sentry;
- Application Insights;
- email-уведомления.

## Примеры

Пример запуска:

```bash
dotnet run
```

Пример команд:

```text
add Buy milk
list
remove Buy milk
remove Unknown task
```

При ошибке удаления несуществующей задачи приложение не падает, а:

1. перехватывает исключение централизованно;
2. выводит сообщение в консоль;
3. записывает детали в `logs/errors.log`.

## Результат

В результате приложение:

- не дублирует `try/catch` по всем операциям;
- ведет единый журнал ошибок;
- хранит полезный диагностический контекст;
- готово к интеграции с внешними системами мониторинга.
